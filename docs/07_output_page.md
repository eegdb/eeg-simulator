# 输出页面 (Output Page)

## 功能概述

输出页面用于配置仿真数据的输出参数，包括采样率、输出格式、持续时间等。还包含仿真控制按钮和状态显示。

## 界面布局

- **上方**：仿真控制区（开始/停止按钮）
- **中间**：输出参数配置
- **下方**：状态信息和进度显示

## 参数详解

### 1. 输出采样率

**参数**：`Sampling Rate` (单位：Hz)

**范围**：100 - 10000 Hz

**默认值**：1000 Hz

**功能**：设置仿真输出的时间分辨率

**意义**：
- 决定时间精度：dt = 1/sampling_rate
- 1000Hz = 1ms时间分辨率
- 更高采样率 = 更精确但更多数据

**推荐设置**：
| 应用场景 | 采样率 | 说明 |
|----------|--------|------|
| 慢波/睡眠 | 200-250 Hz | 满足Nyquist，数据量小 |
| 标准EEG | 500-1000 Hz | 常规分析 |
| ERP研究 | 1000-2000 Hz | 精确潜伏期测量 |
| 高频Gamma | 2000-5000 Hz | >100Hz信号需要 |

**Nyquist定理**：采样率必须 > 2 × 最高频率

---

### 2. 输出格式

**参数**：`Output Format`

**选项**：
| 格式 | 扩展名 | 用途 |
|------|--------|------|
| EDF | .edf | 欧洲数据格式，国际标准 |
| FIF | .fif | MNE原生格式，推荐 |
| CSV | .csv | 纯文本，通用 |
| NumPy | .npy | Python数组，高效 |

#### 2.1 EDF格式

**特点**：
- 国际标准医疗数据格式
- 可被大多数EEG软件读取
- 支持多通道和元数据

**限制**：
- 分辨率限制（16位）
- 文件大小限制（<2GB）

#### 2.2 FIF格式

**特点**：
- MNE工具链原生格式
- 支持完整元数据
- 高效存储（32位浮点）

**推荐用于**：MNE-Python后续分析

#### 2.3 CSV格式

**特点**：
- 纯文本，可读性强
- 可被Excel/Matlab等读取
- 每行一个时间点，每列一个通道

**缺点**：文件体积大

#### 2.4 NumPy格式

**特点**：
- Python数组原生格式
- 读写速度快
- 需用Python读取

---

### 3. 输出时长

**参数**：`Output Duration` (单位：秒)

**范围**：0 - 3600 秒

**默认值**：0（无限）

**特殊值**：
- **0**：无限运行，直到手动停止
- **>0**：运行指定时间后自动停止

**用途**：
- 短时测试：10-60秒
- 标准记录：5-10分钟
- 长时监测：>10分钟

---

### 4. 输出文件路径

**参数**：`Output File`

**功能**：选择保存仿真数据的文件路径

**默认**：项目目录下的`simulation_raw.fif`

**命名建议**：
```
{被试}_{条件}_{日期}_{时长}.fif
例：sub01_resting_20240115_300s.fif
```

---

### 5. 实时显示选项

**复选框**：

| 选项 | 功能 |
|------|------|
| `Save to file` | 保存数据到文件 |
| `Real-time display` | 在Signal页面实时显示波形 |
| `Show progress` | 显示仿真进度条 |

## 仿真控制

### 开始仿真

**按钮**：`Start Simulation`

**操作**：
1. 验证所有参数有效
2. 检查输出路径可写
3. 初始化仿真引擎
4. 启动数据生成和显示

### 停止仿真

**按钮**：`Stop Simulation`

**操作**：
1. 停止数据生成
2. 关闭输出文件
3. 保存最终数据
4. 更新状态为"Stopped"

### 暂停/继续（如果支持）

**按钮**：`Pause / Resume`

**用途**：临时中断而不丢失数据

## 使用流程

### 标准仿真流程

```
1. 设置采样率（根据研究需求）
   ↓
2. 选择输出格式（FIF推荐）
   ↓
3. 设置输出文件路径
   ↓
4. 设置输出时长（或设为0无限）
   ↓
5. 勾选"Save to file"
   ↓
6. 点击"Start Simulation"
   ↓
7. 在Signal页面监控波形
   ↓
8. 达到时长或点击"Stop Simulation"
   ↓
9. 检查输出文件
```

### 测试模式（不保存）

```
1. 取消勾选"Save to file"
   ↓
2. 勾选"Real-time display"
   ↓
3. 点击"Start Simulation"
   ↓
4. 仅实时查看，不生成文件
   ↓
5. 参数满意后再启用保存
```

## 性能考虑

### 磁盘I/O

高采样率+多通道会产生大量数据：
```
数据量(MB/s) = 采样率(Hz) × 通道数 × 4(字节) / 1,000,000

例：1000Hz × 64通道 = 256 kB/s ≈ 15 MB/分钟
```

### 内存使用

实时缓冲区占用：
```
内存(MB) = 通道数 × 缓冲区大小 × 8(字节) / 1,000,000

例：64通道 × 5000点 × 8 = 2.56 MB
```

## 故障排除

| 问题 | 原因 | 解决 |
|------|------|------|
| 无法开始仿真 | 未选择通道 | 在Channels页面选择通道 |
| 文件写入失败 | 路径无效/权限不足 | 检查路径和权限 |
| 磁盘空间不足 | 长时间仿真 | 缩短时长或清理磁盘 |
| 帧率下降 | 采样率过高 | 降低采样率或减少通道 |
| 数据不连续 | I/O瓶颈 | 关闭实时显示或使用SSD |

## 相关概念

### 数据精度

- **16位**：EDF格式，约0.1μV分辨率
- **32位浮点**：FIF格式，高精度
- **64位双精度**：内部计算使用

### 文件大小估算

```
文件大小(MB) = 采样率 × 通道数 × 时长(秒) × 4 / 1,000,000

例：1000Hz × 64通道 × 300秒 = 76.8 MB
```

### 批处理 vs 实时

- **实时模式**：边生成边保存（当前实现）
- **批处理模式**：先生成全部再保存（未来功能）

实时模式内存占用小，适合长时记录。
